一、爆炸根因链 (辩论+验证确认)
═══════════════════════════════════════════════════════════════

  已验证排除的假说:
  ❌ gate饱和导致永久冻结 → 验证: gate饱和率最高51%, 未全饱和
  ❌ aux_s阶跃导致gate突变 → 验证: aux_s对gate输出零影响(diff=0.0000)
  ❌ explore无tanh导致二级放大 → 验证: tanh有无gate输出几乎相同
  ❌ meta sigmoid梯度消失 → 验证: gate输入在0.5-2.0区间, 梯度充足

  确认的爆炸链:
  模型收敛(loss~0.0001) → 数据噪音触发单次CE spike
  → spike检测窗口太窄(需连续2轮上升) → 未触发保护
  → CE梯度暴力改变P7权重 → P7输出畸变
  → ABC收到畸变输入 → ac_batch指数增长
  → aux_scale被cool_down压到0.02 → ABC辅助梯度断开
  → 但CE主梯度仍在破坏P7 → 不可逆崩溃
  → EMA被CE spike推到天文数字 → 数值溢出 → NaN

  核心教训:
  1. 单次spike检测比连续趋势检测更有效
  2. 主火(CE)必须和辅火(ABC)同时灭
  3. 收敛后应主动停车, 不跑无用轮

二、5层防护体系
═══════════════════════════════════════════════════════════════

  第1层: ac_exploding 实时检测 (gate之前)
    触发: ac_batch.norm > ac_ema × 3
    动作: cool_down=30, 降低aux_scale
    优势: 第一时间, 不等CE

  第2层: CE spike 单次检测 (CE之后)
    触发: ce > max(ema×3, 5.0)
    动作: cool_down=50, 当批CE clamp到5.0
    优势: 单次即触发, 不等连续两轮

  第3层: aux_scale 渐进自愈
    跌: aux_val *= 0.95 (5%/步, 快速响应)
    涨: aux_val *= 1.05 (5%/步, ~20步恢复到1.0)
    优势: 无阶跃, 无50倍突变

  第4层: NaN自动回滚
    触发: 任意参数梯度含NaN
    动作: 跳过optimizer.step() → 从best.pt恢复 → 清空梯度
    优势: 炸不废, 自动从上一个好检查点继续

  第5层: 收敛自动停车
    触发: avg_loss < 0.0001
    动作: 保存并退出训练
    优势: 不跑无用轮, 防止过拟合后噪音破坏权重

三、渐进学习率
═══════════════════════════════════════════════════════════════

  每个优化器独立追踪loss趋势:
    本批loss < EMA → lr × 1.005 (加速)
    本批loss > EMA → lr × 0.995 (减速)
    lr范围: [1e-6, 0.1]

  7路独立:
    p7_in: base×1.0, p7: base×1.5, p6: base×1.0
    p3l: base×0.8, abc: base×1.2, gate: base×2.0
    master: base×0.5

五、双核辩论总结
═══════════════════════════════════════════════════════════════

  辩论1: gate叠加问题
    问题: gate_input加法导致ABC被prev_loss淹没
    结论: 验证显示aux_s对gate输出零影响, 加法不是问题
    修复: abc_to_gate/sent_to_gate小随机初始化(std=0.01/0.1)

  辩论2: ABC颗粒度对齐
    问题: 64D→128D→256D?
    结论: 128D最优, 3层126槽
    实施: P3扩到128D, Layer1-3分层

  辩论3: 爆炸根因
    初始假说: gate饱和/aux阶跃/explore无tanh
    验证推翻: gate饱和51%max, aux对gate零影响, tanh无差异
    真实根因: CE单次spike检测失效 + CE主梯度无约束
    修复: 单次spike检测 + CE clamp + 渐进aux + NaN回滚
