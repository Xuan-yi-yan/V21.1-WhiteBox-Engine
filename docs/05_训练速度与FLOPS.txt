  V21 训练速度参考 (RTX 5070 12GB, 16M参数, FP32)
═══════════════════════════════════════════════════════════════

一、速度公式

  每轮时间 ≈ (数据量 / batch_size) × 单批耗时

  单批耗时 ≈ 0.9s (P7交叉注意力占大头, nA×3000词表)
  开amp(混合精度) → 单批 ~0.6s

二、不同数据量预估

  ┌──────────┬──────────┬────────────┬──────────┬─────────────┐
  │ 数据量   │ batch=14 │ batch=32   │ batch=56 │ 显存占用    │
  ├──────────┼──────────┼────────────┼──────────┼─────────────┤
  │ 15对     │ ~1s/轮   │ ~1s/轮     │ ~1s/轮   │ 3-4GB      │
  │ 150对    │ ~10s/轮  │ ~5s/轮     │ ~3s/轮   │ 4-5GB      │
  │ 1500对   │ ~90s/轮  │ ~45s/轮    │ ~25s/轮  │ 6-7GB      │
  │ 5000对   │ ~5min/轮 │ ~2.5min/轮 │ ~1.5min/轮│ 7-8GB     │
  │ 52000对  │ ~50min/轮│ ~25min/轮  │ ~14min/轮 │ 9-10GB    │
  └──────────┴──────────┴────────────┴──────────┴─────────────┘

  当前: 52K对, batch=32, ~25min/轮, 显存~9GB
  1500轮总耗时: 1500×25min = 625小时 ≈ 26天

四、加速选项

  torch.amp('cuda'): 已开启 ✅ → 单批 0.9s→0.6s
  batch加大: 32→56 → 25min/轮→14min/轮 (但显存+2GB)
  关桌面应用: 释放3-5GB → 可开batch=56
  torch.compile: Windows无Triton, 不推荐 ❌

五、单字符FLOPS精确计算 (V21白盒, 16M参数)

  模型架构: CharEmbed(256D)→P7(16头×16D,全词表K/V)→ABC→Gate(384D)→P6(128头Linear)

  ┌──────────────────────────┬─────────────────┬──────────┐
  │ 组件                     │ 每批FLOPS       │ 占比     │
  ├──────────────────────────┼─────────────────┼──────────┤
  │ P7 K投影(vocab×32)       │ 12.58G          │ 36.4%    │
  │ P7 V头(vocab×32)         │ 12.58G          │ 36.4%    │
  │ P7 QK^T                  │ 0.79G           │ 2.3%     │
  │ P7 Attn×V                │ 0.79G           │ 2.3%     │
  │ P7 Q投影+组竞争+out      │ 0.14G           │ 0.4%     │
  │ p7in+ABC+Gate            │ 0.49G           │ 1.4%     │
  │ P6(128头+tied logits)    │ 6.90G           │ 20.0%    │
  │ 其他(P3-L, CE loss等)    │ 0.30G           │ 0.9%     │
  ├──────────────────────────┼─────────────────┼──────────┤
  │ 每批总计 (batch=32)      │ 34.57G          │ 100%     │
  │ 单字符 (÷512字/batch)    │ 67.5M           │          │
  └──────────────────────────┴─────────────────┴──────────┘

  关键发现:
    - P7的K/V投影对全词表(3000字)做, 但batch内每样本重复算一遍
      → 32次×3000字×256D = 25.2G/批浪费 (可优化: 算1次广播)
    - P6的128头用Python for循环, 每头单独Linear(256,256)
      → 128次kernel launch, 每次算4.2M FLOPS, launch开销远大于计算
    - 有效计算(去冗余): ~8.5G/批 → 单字符 ~16.6M FLOPS

  每轮总FLOPS: 34.57G × 1625批 ≈ 56.2T

  GPU利用率分析:
    RTX 5070 FP16 tensor core峰值: ~45 TFLOPS
    理论每轮: 56.2T ÷ 45T ≈ 1.2s  ← 纯计算
    实际每轮: ~1500s (25min)        ← 差距1250×

    差距原因:
    ① P7 attention: [32,16,16,3000] softmax + matmul, 非密集矩阵乘
       内存带宽瓶颈, 非计算瓶颈
    ② P6 for循环128头: 每头独立kernel launch, Python开销 >> GPU计算
    ③ PyTorch eager mode: 每个op都是独立kernel, 无fusion
    ④ 数据加载: CPU P3属性处理 + padding (每批~0.1s)
    ⑤ 优化器step: 7个Adam, 16M参数更新 (每批~0.05s)

    有效GPU利用率: ~0.08% ← 架构特性, 非bug
    (attention-heavy + 小batch + Python loop = GPU大量空闲等kernel)

  优化方向:
    - K/V投影去重: 25.2G/批 → 0.8G/批 (节省73%无用计算)
    - P6 128头合并为单次matmul: batch×128×256 @ 256×V
      消除128次kernel launch → P6速度提升10-50×
    - torch.compile (Windows需Triton → Linux可行)
    - 当前架构下加大batch到56是最实际的加速手段 ✅

六、实时判定

  看进度条 ETA 即可。52K≈1625批/轮, 每批~0.92s → 25min/轮。
  跑100轮 ≈ 42小时。跑1500轮 ≈ 26天。
